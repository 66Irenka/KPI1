Контрольні питання з CSS

    Для чого використовуються каскадні таблиці стилів?

    У моєму проєкті інтерактивного каталогу книг каскадні таблиці стилів (CSS) застосовуються для оформлення всього інтерфейсу: шапки сайту, головного меню, блоку пошуку, таблиці топ-книг, галереї, блогу, форми зворотного зв’язку та модального вікна.

    CSS дозволяє відокремити структуру (HTML) від зовнішнього вигляду, а також гнучко керувати стилями. Наприклад, перемикання теми сайту (темна/світла) реалізоване через зміну CSS-змінних без зміни HTML.

  1. Які методи визначення стилів у HTML-документі ви знаєте?

    Існує три основні методи написання CSS:

    • Inline-стилі — стилі в атрибуті style:
      <button style="color:white;">OK</button>
    • Внутрішні стилі — у блоці <style>:
      <style> h1 { color:red; } </style>
    • Зовнішні стилі — у файлі style.css, підключеному так:
      <link rel="stylesheet" href="style.css">

    У моєму проєкті використовується *зовнішній CSS-файл*, який містить оформлення всіх компонентів інтерфейсу.

  2. У чому різниця між каскадуванням і спадкуванням?

    Каскадування — це механізм, за яким браузер вибирає кінцевий стиль, коли до одного елемента застосовано кілька різних CSS-правил. У моєму проєкті це особливо помітно під час перемикання теми сайту: нові значення CSS-змінних перекривають попередні згідно з пріоритетом каскаду.

    Спадкування — властивості тегів (наприклад, font-family, color, line-height) передаються дочірнім елементам автоматично. У моєму каталозі всі блоки (меню, блог, модальне вікно) успадковують шрифт Montserrat, заданий у body.

  3. Як оформлюються селектори тегу і класу?

    Селектор тегу застосовується до всіх елементів певного типу:

    p {
    line-height: 1.6;
    color: #444;
}

    Селектор класу застосовується лише до елементів з певним класом:

    .blog-card {
    background-color: #ffffff;
    border-radius: 12px;
    padding: 20px;
}

    Приклад із галереї:

    .gallery-image {
    width: 100%;
    border-radius: 12px;
    object-fit: cover;
}

  4. Для чого використовується ідентифікатор?

    Ідентифікатор id використовується для елементів, що мають бути унікальними на сторінці.

    У моєму проєкті він застосовується для:

    • #root — контейнер для результатів пошуку книг
    • #scrollTopBtn — кнопка швидкої прокрутки вгору
    • #bookModal — модальне вікно з детальною інформацією
    • #episodeSelect — select зі списком знайдених книг

    Приклад HTML:

    <div id="bookModal" class="modal-backdrop">
    ...
</div>

    Приклад CSS:

    #scrollTopBtn {
    position: fixed;
    bottom: 20px;
    right: 20px;
}

  5. Навести приклади оформлення текстових елементів.

    У моєму каталозі текстове оформлення використовується для заголовків, меню, блогу та футера.

    body {
    font-family: "Montserrat", sans-serif;
}

/* Заголовок у шапці */
.app-header h1 {
    font-size: 32px;
    font-weight: 700;
}

/* Підзаголовок блогу */
.blog-subtitle {
    font-size: 16px;
    color: #6b7280;
}

/* Посилання в головному меню */
.main-nav a {
    font-weight: 500;
    text-decoration: none;
}

/* Текст у футері */
.footer-contact p {
    font-size: 14px;
    color: #4b5563;
}

    Такі стилі забезпечують читабельність, акуратність та єдиний стиль інтерфейсу.
